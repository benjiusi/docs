# How BlankFX works

BlankFX is a purpose-built FX protocol. Unlike general-purpose DEXs that treat every token pair the same, BlankFX separates currency management from swap execution.

## The two-layer model

```
                    ┌─────────────────────┐
                    │   BlankFX Router     │
                    │   (swap entry point) │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
     ┌────────▼────────┐  ┌───▼────┐   ┌───────▼───────┐
     │    USD Hub       │  │EUR Pool│   │  Future Pools  │
     │ USDC·USDT·RLUSD │  │  EURC  │   │  CHF·GBP·...  │
     └─────────────────┘  └────────┘   └───────────────┘
```

**Layer 1 — USD Hub:** Holds all USD-denominated stablecoins (USDC, USDT, RLUSD) in a single pool. Swaps between USD stablecoins happen here at near 1:1 rates.

**Layer 2 — FX Pools:** Each foreign currency gets its own pool. The EUR Pool holds EURC, a future CHF Pool would hold CHFAU, and so on. Cross-currency swaps route through the USD Hub and the relevant FX Pool using oracle rates.

## Swap routing

The Router determines the path automatically based on your token pair:

| Swap | Route | Example |
|------|-------|---------|
| USD → USD | Direct through USD Hub | USDC → USDT |
| USD → Foreign | USD Hub → FX Pool | USDC → EURC |
| Foreign → USD | FX Pool → USD Hub | EURC → USDT |
| Foreign → Foreign | FX Pool → USD Hub → FX Pool | EURC → GBPT (future) |

Every swap is atomic. Either the full trade completes or nothing happens. No partial fills, no stuck funds.

## Oracle pricing

Instead of deriving prices from pool ratios (which creates arbitrage opportunities and MEV), BlankFX reads FX rates from on-chain oracle feeds.

The oracle rate is the reference point. The protocol applies a small spread on top, and the result is the rate you see in the interface.

This means:
- Large trades don't move the price against you
- The rate is anchored to real-world FX markets
- No sandwich attacks or frontrunning on your trade

## Settlement flow

```
1. You submit a swap (or sign a gasless order)
2. Router checks your access tier
3. Router reads the oracle rate for the currency pair
4. Router calculates output amount minus spread
5. Your input tokens move to the source pool
6. Output tokens move from the destination pool to your wallet
7. Done — single transaction, single block
```

Settlement is final the moment the transaction confirms. No pending states, no clearing period.

## Access control

BlankFX uses a tiered access system managed by the BlankFXAccess contract. Users must be registered to trade. Access tiers can limit maximum trade sizes and determine which features are available.

This exists for compliance reasons. As an FX protocol serving institutional users, BlankFX needs to know its counterparties.
